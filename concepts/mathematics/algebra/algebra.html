Algebra

In arithmetic you work with particular numbers: 2 + 3 = 5, 4 + 1 = 5. Algebra is the next step — instead of one fixed number, we use a letter to stand for any number. A letter like a or n is a placeholder: it holds a spot that any number could fill.

That one idea is surprisingly powerful. It lets us write a single statement that captures a whole pattern at once, instead of checking number after number forever. It also lets us describe a number we don't know yet — the amount of pocket money left after buying a snack, the number of seats still empty on a bus, the score you need on your last test to reach an average — without knowing its value in advance. A letter is a box with a number-sized hole in it, waiting to be filled in.

We already met a fact about the laws of arithmetic: swapping the order of an addition never changes the answer. With numbers we can only ever show examples — 2 + 3 = 3 + 2, 4 + 1 = 1 + 4, and so on. But there are infinitely many pairs, so no matter how many examples we write down, we can never quite finish the list and be completely sure the pattern never breaks.

Algebra lets us say it once, for every pair at the same time, and be completely sure — because a letter doesn't mean "some particular number I haven't told you," it means "put any number here and the statement stays true." Press play to watch two concrete number facts fade into letters and become a single general rule that covers all of them at once:

a + b = b + a

Notice what just happened to the numbers 2 and 3 in the animation: they didn't turn into "the letter version of 2 and 3" — they turned into a and b, which no longer mean 2 and 3 specifically. Once we generalise, the letters are free to represent absolutely any pair of numbers you like: 10 and 1000, \tfrac{1}{2} and -7, or even two numbers nobody has ever written down before. The rule doesn't care — it works for all of them, forever, because of how it was built.

Worked example: turning words into algebra

A huge part of algebra is translation — taking an everyday sentence and rewriting it using letters and symbols. Try this one:

"Think of a number. Add 5 to it."

We don't know the starting number, so give it a letter — say n. "Add 5 to it" means the answer is 5 more than n, which we write as:

n + 5

That short expression means exactly the same thing as the whole sentence, for every starting number at once. If you want the answer for one particular number, you substitute it in for n — that is, you replace the letter with that number everywhere it appears. Suppose the number you thought of was 7: replace n with 7 and work it out as ordinary arithmetic:

n + 5 = 7 + 5 = 12

Here's the same idea as a tiny program — the function does exactly what the expression n + 5 says, and we can ask it for the answer for as many starting numbers as we like without rewriting anything:

function fivePlus(n: number): number { return n + 5; } console.log(fivePlus(7)); // 5 more than 7 console.log(fivePlus(20)); // 5 more than 20 console.log(fivePlus(0)); // 5 more than 0

Words that translate into algebra turn up again and again with slightly different phrasing — "more than", "less than", "double", "half of", "times" — but they all boil down to the same handful of operations you already know from arithmetic:

  • "8 more than a number" → n + 8
  • "5 less than a number" → n - 5
  • "double a number" → 2n
  • "half of a number" → \dfrac{n}{2}

These two catch nearly everyone the first time they meet algebra:

  • A letter always means the same number, everywhere it appears in one expression or problem. In n + n, both copies of n are the same number — you can't let the first n secretly be 3 and the second one be 7. If n = 3, then n + n = 3 + 3 = 6, never anything else, for as long as n keeps that value in that problem.
  • 2n means 2 \times n. In algebra we almost always drop the multiplication sign between a number and a letter (and between two letters, as in ab for a \times b) — writing 2 \times n in full gets messy fast, because \times looks so much like the letter x. So 2n is not "the digits 2 and n stuck together" the way 25 means twenty-five — it's a hidden multiplication, and it's worth saying "two times n" in your head every time you read it, until the habit sticks.

Worked example: collecting like terms

Once a phrase becomes an expression, algebra lets you simplify it — rewrite it in a shorter, tidier form that means exactly the same thing. The most common way to simplify is collecting like terms: combining pieces of the expression that are built from the same letter.

Think about 3n + 2n. Whatever number n turns out to be, 3n means "three lots of n" and 2n means "two lots of n." Three lots of something plus two lots of the same something is always five lots of it — exactly the way 3 apples and 2 apples make 5 apples, without needing to know what an "apple" weighs. So:

3n + 2n = 5n

This isn't a coincidence that happens to work for one value of n — it's true for every value, because we never used what n actually was, only that both terms were "lots of n." Check it with the code below: the unsimplified and simplified versions give the same answer for every number you try.

function unsimplified(n: number): number { return 3 * n + 2 * n; } function simplified(n: number): number { return 5 * n; } console.log(unsimplified(4), simplified(4)); // 20 20 console.log(unsimplified(10), simplified(10)); // 50 50 console.log(unsimplified(-2), simplified(-2)); // -10 -10

Terms like 3n and 2n — built from exactly the same letter — are called like terms, and only like terms can be combined this way. 3n and 5 can't be collected into one term (three lots of an unknown number plus a plain five isn't "eight of anything" in general), and 3n and 3x can't either, unless you happen to know that n and x are equal. So an expression like 2n + 3 + n simplifies by collecting only the n terms together, leaving the plain number alone:

2n + 3 + n = 3n + 3

Around 820 AD, a Persian mathematician and astronomer named Muhammad ibn Musa al-Khwarizmi, working at the House of Wisdom in Baghdad, wrote a book with a title that translates roughly as "The Compendious Book on Calculation by Completion and Balancing." The Arabic word for "completion" in that title was al-jabr — and it referred to a very specific technique: if an equation had a term being subtracted, you could "complete" or "restore" it by adding that amount to both sides, so nothing was left in a negative, unbalanced state.

Centuries later, when the book was translated into Latin and spread across medieval Europe, that one word — al-jabr — got carried over almost unchanged, eventually settling into the English spelling we still use today: algebra. (His own name, al-Khwarizmi, went on its own journey through Latin translation too — it's where the word "algorithm" comes from.) So every time you "balance" an equation by doing the same thing to both sides, you're using a technique — and a name — that has been travelling the world, mostly unchanged, for over 1,200 years.

And the reach of that one idea — a letter standing for any number — is hard to overstate. A video game doesn't store a separate rule for every possible sword hit; it stores one algebraic expression for damage, something like d = (a - r) \times m, where a is your attack strength, r is the enemy's defence, and m is a critical-hit multiplier — then substitutes in whatever numbers apply to that particular hit, millions of times a second. Al-Khwarizmi never saw a video game, but he'd recognise the trick instantly.

Once a pattern is written with letters, we can do things with it. Algebra is the toolkit for working with these general statements: we can apply the order of operations to expressions full of letters, simplify them by collecting like terms (as we just did), solve equations to find an unknown number, and even draw expressions as graphs. The same letters happily stand for fractions or negative numbers too — a rule like a + b = b + a holds for every kind of number, not just the counting numbers.
